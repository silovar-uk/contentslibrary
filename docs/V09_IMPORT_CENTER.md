# v0.9 データ取込センター

## 目的

大量の作品データを、いきなり本番の作品一覧へ追加せず、owner専用のステージング領域で確認してから反映する。

旧GitHub Actions経由の暗号化投入は廃止した。取込経路はアプリの設定画面にある「データ取込センター」だけとする。

## 利用条件

- Cloudflare Accessでログイン済み
- アプリ上のroleが`owner`
- 取込センターを明示的に有効化している

有効時間は60分。反映完了時には自動で無効になる。

## 取込の流れ

1. 設定とデータを開く
2. データ取込センターを60分間だけ有効化
3. JSONファイルを選択
4. ブラウザ上で作品数・メモ数・SHA-256を確認
5. ステージングへ送る
6. 検証結果を確認
7. 競合が0件の場合だけ本番へ反映
8. 必要に応じて、今回の取込だけを取り消す

ファイル選択、アップロード、検証の段階では、通常の作品一覧は変更されない。

## JSON形式

```json
{
  "version": 1,
  "batch": "kindle-notion-2026-07-20",
  "generated_at": "2026-07-20T00:00:00.000Z",
  "items": [
    {
      "source_key": "kindle:example",
      "title": "作品名",
      "creator": "作者名",
      "type": "book",
      "status": "owned_unread",
      "short_note": null,
      "labels": {
        "genre": ["小説"],
        "theme": [],
        "tag": ["Kindle購入済み"]
      },
      "metadata": {
        "source": "kindle"
      },
      "created_at": "2026-07-20T00:00:00.000Z",
      "updated_at": "2026-07-20T00:00:00.000Z",
      "notes": [
        {
          "note_type": "quote",
          "content": "引用またはメモ",
          "position": null,
          "created_at": "2026-07-20T00:00:00.000Z",
          "updated_at": "2026-07-20T00:00:00.000Z"
        }
      ]
    }
  ]
}
```

### 必須項目

- `items`
- 各作品の`source_key`
- `title`
- `type`
- `status`

### 対応する作品種別

- `book`
- `manga`
- `movie`
- `anime`
- `drama`
- `other`

### 対応する状態

- `want`
- `owned_unread`
- `active`
- `completed`
- `paused`
- `dropped`

## 検証ルール

### スキップ

既存作品と`source_key`が一致する場合。再実行による二重登録を防ぐ。

### 既存作品へ統合

作品種別と正規化した作品名が一致し、既存作品が1件だけ存在する場合。

既存の評価、状態、一言メモ、作者などは上書きしない。取込ファイルの分類とメモだけを追加する。

### 新規追加

`source_key`も作品名も既存作品と一致しない場合。

### 競合

- 取込ファイル内で`source_key`が重複
- 同じ作品種別・作品名が取込ファイル内で重複
- 既存データ側に同名作品が複数存在
- 事前申告した作品数・メモ数と実データが不一致

競合が1件でもある場合、本番反映はできない。

## 分割処理

- ブラウザからステージングへ送る単位：25作品
- APIが受け付ける最大単位：50作品
- 本番へ反映する単位：20作品
- 取消する単位：100変更

通信や画面操作が途中で止まっても、同じファイルを選んで続きから再開できる。

## 取消

反映時に追加した次の情報を記録する。

- 新規作品
- 新規メモ
- 作品と分類の新しい紐付け

取消では、この記録を逆順に処理する。

- 新規作品はソフト削除
- 新規メモは削除
- 今回追加した分類の紐付けだけを解除

取込前から存在した作品の評価、状態、一言メモなどは変更しないため、取消でも影響を受けない。

## セキュリティ

- owner以外はAPIを含め利用不可
- 変更APIは同一オリジン確認が必須
- 取込操作は60分間だけ有効
- 本番反映と取消には確認文字の入力が必要
- 公開GitHubリポジトリへ作品一覧、暗号断片、秘密鍵を保存しない
- 旧一括投入ワークフロー、復号スクリプト、一時公開鍵は削除済み
